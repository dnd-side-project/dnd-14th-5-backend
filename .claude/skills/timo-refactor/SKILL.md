# Timo 안전 리팩토링

대상: $ARGUMENTS

## Phase 1: 병렬 사전 분석

**에이전트 A — 테스트 커버리지:**
- `src/test/`에서 대상 관련 테스트 파일 전부 탐색
- 현재 테스트가 커버하는 동작 목록화
- 리팩토링 후 깨질 가능성 있는 테스트 식별

**에이전트 B — 영향 범위:**
- 대상 클래스/메서드를 참조하는 모든 파일 탐색
- 인터페이스 계약 변경 여부 판단
- DB/API 입출력 구조 변경 여부 판단

## 판단 기준

입출력·DB 구조가 변하지 않는 경우만 진행.
변한다면 **중단**하고 영향 범위를 먼저 보고.

## Phase 2: 리팩토링 실행

1. 테스트가 커버하지 않는 동작 → 먼저 테스트 추가
2. 리팩토링 수행
3. Timo 컨벤션 정리:
   - 불필요한 주석 제거
   - Java 관용 패턴 (`Optional`, `Stream`, `var`) 적용
   - `record` 전환 가능한 DTO 전환
   - 소프트 딜리트 쿼리 `AndDeletedAtIsNull` suffix 통일
   - 미사용 import 제거

## Phase 3: 검증

Skill 도구로 `validate-before-commit` 실행

## 결과 리포트

```
### 변경 범위
- 수정 파일: N개
- 입출력 변경: 없음 ✅
- DB 구조 변경: 없음 ✅

### 테스트 상태
- 기존 테스트: 통과 ✅
- 추가 테스트: N개
```
